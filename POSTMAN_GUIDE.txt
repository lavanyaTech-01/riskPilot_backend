# Postman POST Mapping Format for Saving Scans to History

## 📌 Save File Analysis Endpoint

### Endpoint Details
```
Method: POST
URL: http://localhost:8080/api/history/save/file
Authentication: Bearer Token (JWT)
Content-Type: application/json
```

---

## 🔧 Postman Setup - Step by Step

### 1. Set Request Method
- Select: **POST**
- URL: `http://localhost:8080/api/history/save/file`

### 2. Add Headers Tab
| Key | Value |
|-----|-------|
| Authorization | Bearer {YOUR_JWT_TOKEN} |
| Content-Type | application/json |

### 3. Add Body (Raw JSON)

```json
{
  "fileName": "document.pdf",
  "fileType": "application/pdf",
  "response": {
    "riskLevel": "HIGH",
    "trustScore": 7.5,
    "email": "hr@company.com",
    "url": "https://suspicious-link.com",
    "emailVerified": false,
    "companyVerified": true,
    "companyDetails": "Company XYZ Ltd",
    "analysisSummary": "This document contains several red flags indicating potential scam content",
    "suspiciousIndicators": [
      "Unsolicited job offer",
      "Requests for personal information",
      "Suspicious email domain"
    ],
    "suggestions": [
      "Do not respond to unsolicited messages",
      "Verify company through official website",
      "Report suspicious activity"
    ]
  }
}
```

## 📝 Request Body Field Details

### For File Analysis (POST /api/history/save/file)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| fileName | string | ✅ Yes | Name of the file analyzed (e.g., "document.pdf") |
| fileType | string | ❌ No | MIME type (e.g., "application/pdf", "image/png") |
| response | AiAnalysisResponseDto | ✅ Yes | Complete analysis response from /api/scan/file |

### Response Fields in AiAnalysisResponseDto

| Field | Type | Description |
|-------|------|-------------|
| riskLevel | string | HIGH, MEDIUM, or LOW |
| trustScore | double | Score 0-10 (0=highly suspicious, 10=trusted) |
| email | string | Extracted email from content |
| url | string | Extracted URL from content |
| emailVerified | boolean | Whether email is from verified domain |
| companyVerified | boolean | Whether company was identified |
| companyDetails | string | Details about the company |
| analysisSummary | string | Summary of the analysis |
| suspiciousIndicators | array | List of detected scam indicators |
| suggestions | array | List of recommendations |
| reviewAnalysis | object | Optional: Online review analysis |

## 🔐 Authentication

### Getting JWT Token First

```
POST http://localhost:8080/api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```

Response:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"
}
```

Copy the `token` value and use it in the Authorization header:
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 💾 Complete Workflow in Postman

### 1. First Request - Login (Get JWT)
```
Method: POST
URL: http://localhost:8080/api/auth/login
Body (raw JSON):
{
  "email": "user@example.com",
  "password": "password123"
}
```

Save the token from response

### 2. Second Request - Scan File
```
Method: POST
URL: http://localhost:8080/api/scan/file
Headers:
  Authorization: Bearer {token_from_step_1}
Body (form-data):
  file: [select your file]
```

Save the response (analysis result)

### 3. Third Request - Save to History
```
Method: POST
URL: http://localhost:8080/api/history/save/file
Headers:
  Authorization: Bearer {token_from_step_1}
  Content-Type: application/json
Body (raw JSON):
{
  "fileName": "{filename from step 2}",
  "fileType": "{file content type from step 2}",
  "response": {paste entire response from step 2}
}
```

Response:
```json
{
  "success": true,
  "message": "File analysis saved to history",
  "historyId": 42
}
```

### 4. Fourth Request - View Saved History
```
Method: GET
URL: http://localhost:8080/api/history/42
Headers:
  Authorization: Bearer {token_from_step_1}
```

Response will show all saved details including deserialized response!

## ✨ Quick Reference

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/scan/file` | POST | Analyze file (returns response) |
| `/api/history/save/file` | POST | Save file analysis to history |
| `/api/scan/url` | POST | Analyze URL (returns response) |
| `/api/history/save/url` | POST | Save URL analysis to history |
| `/api/scan/email` | POST | Analyze email (returns response) |
| `/api/history/save/email` | POST | Save email analysis to history |
| `/api/scan/description` | POST | Analyze description (returns response) |
| `/api/history/save/description` | POST | Save description analysis to history |
| `/api/history/all` | GET | View all history |
| `/api/history/{id}` | GET | View detailed history entry |
