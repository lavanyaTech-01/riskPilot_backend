# POST /api/history/save/file - Updated Format

## ✅ NEW: File in Form-Data + Response as Form Parameter

The endpoint now accepts **BOTH file and response in form-data format** - no raw JSON needed!

---

## 📋 Postman Setup

### Endpoint
```
Method: POST
URL: http://localhost:8080/api/history/save/file
```

### Headers
```
Authorization: Bearer {JWT_TOKEN}
Content-Type: multipart/form-data (auto-set by Postman)
```

### Body - Select form-data
```
Key         | Type   | Value
─────────────────────────────────────────────────────
file        | File   | [Select your file] 📁
response    | Text   | [Paste JSON here]
fileName    | Text   | document.pdf (optional)
fileType    | Text   | application/pdf (optional)
```

---

## 🎯 How It Works

### BEFORE (Raw JSON):
```json
POST /api/history/save/file
Content-Type: application/json

{
  "fileName": "document.pdf",
  "fileType": "application/pdf",
  "response": { ...JSON... }
}
```

### NOW (Form-Data):
```
POST /api/history/save/file
Content-Type: multipart/form-data

file:      [binary file content]
fileName:  document.pdf
fileType:  application/pdf
response:  {"riskLevel":"HIGH", ...JSON...}
```

## 📝 Step-by-Step Instructions

### Step 1: Get JWT Token
```
POST http://localhost:8080/api/auth/login
{
  "email": "user@example.com",
  "password": "password123"
}

Response: {"token": "eyJhbGc..."}
```

### Step 2: Prepare Your Response JSON
First, scan a file to get the response:

```
POST http://localhost:8080/api/scan/file

Response:
{
  "riskLevel": "HIGH",
  "trustScore": 2.3,
  "email": "hr@company.com",
  "url": "https://suspicious.com",
  "emailVerified": false,
  "companyVerified": false,
  "analysisSummary": "Multiple scam indicators",
  "suspiciousIndicators": ["Unsolicited offer"],
  "suggestions": ["Do not respond"]
}

💾 COPY THIS ENTIRE JSON
```

### Step 3: Save to History (Form-Data)
```
1. Create new Postman request
2. Method: POST
3. URL: http://localhost:8080/api/history/save/file
4. Authorization tab → Bearer Token → paste JWT
5. Body tab → select form-data
6. Add form fields:

   Key: file
   Type: File
   Value: [Click Select Files and choose your file]

   Key: response
   Type: Text
   Value: [Paste the JSON response from Step 2]

   Key: fileName
   Type: Text
   Value: JobOffer.pdf

   Key: fileType
   Type: Text
   Value: application/pdf

7. Click Send!
```

---

## 📊 Form-Data Parameters Explained

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| file | File | No | Actual file upload (optional, but recommended) |
| response | Text/JSON | Yes | Complete analysis response as JSON string |
| fileName | Text | No | File name (auto-detected from file if not provided) |
| fileType | Text | No | MIME type (auto-detected from file if not provided) |

---

## 🎯 Key Points

✅ **Use form-data** (not raw JSON)
✅ **response parameter** should be entire JSON as string
✅ **file parameter** is optional but recommended to upload actual file
✅ **fileName and fileType** are optional (auto-detected if not provided)
✅ **Authorization** header required with JWT token

## 💡 Real Example

### Postman Body (form-data):
```
file: [select: JobOffer.pdf]

response: {"riskLevel":"HIGH","trustScore":2.3,"email":"hr@company.com","url":"https://suspicious.com","emailVerified":false,"companyVerified":false,"analysisSummary":"Multiple scam indicators","suspiciousIndicators":["Unsolicited offer"],"suggestions":["Do not respond"]}

fileName: JobOffer.pdf

fileType: application/pdf
```

### Request Breakdown:
```
POST /api/history/save/file HTTP/1.1
Host: localhost:8080
Authorization: Bearer eyJhbGc...
Content-Type: multipart/form-data; boundary=----WebKit...

------WebKit...
Content-Disposition: form-data; name="file"; filename="JobOffer.pdf"
Content-Type: application/pdf

[binary file content here]
------WebKit...
Content-Disposition: form-data; name="response"

{"riskLevel":"HIGH","trustScore":2.3,...}
------WebKit...
Content-Disposition: form-data; name="fileName"

JobOffer.pdf
------WebKit...
Content-Disposition: form-data; name="fileType"

application/pdf
------WebKit...--
```

### Response:
```json
{
  "success": true,
  "message": "File analysis saved to history",
  "historyId": 42
}
```

## 🔍 Error Responses

### Missing Response
```json
{
  "success": false,
  "error": "Analysis response is required"
}
```

### Missing File Name
```json
{
  "success": false,
  "error": "File name is required"
}
```

### Invalid JSON in Response
```json
{
  "success": false,
  "error": "Invalid JSON in response parameter: ..."
}
```

### Unauthorized
```json
{
  "success": false,
  "error": "Unauthorized - Invalid or missing JWT token"
}
```

## ✅ Advantages of This Format

| Advantage | Benefit |
|-----------|---------|
| Single endpoint | No need for two-step process |
| Form-data throughout | Consistent format |
| File upload included | Can store actual file bytes |
| Flexible | Response can be provided as string |
| Simple | No JSON parsing complexity |
